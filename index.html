Git

Gitflow

  1. Start with Gitbash
  2. Navigate to bootcamp > prework study guide
  3. Check status
  4. Create branch using 'checkout -b branchname'
  5. Check status
  6. Add staged files to new/working branch(your changes) using 'add -A'
  7. Commit the changes to new/working branch using commit -m
  8. Make sure you're using the most updated main branch (or dev branch depending where you commit to) using 'pull origin main' (or dev depending where you are committing to)
  9. Push your changes to the new/working branch using 'push origin new/workingbranchname'
  10. Navigate to GitHub and use the Compare and Pull request button
  11. Make sure the status bar is showing Main (or your branch you are merging with) and your new/working branch
  12. Create a Pull Request(PR)
  13. Ready for review and feedback
  14. Once approved, it is then merged.
  15. Once merged, perform a 'pull origin main' to be sure you're up-to-date on your code in VS
  16. moe note: If you do a pull origin main, then you have a recent copy. However before you start coding again, check the status and make sure you have the most recent version.
  17. before you start coding, make sure you have a new branch you will be working with and also do the "add -A" so it initializes you making changes to this new branch.
  18. you always have to do a "add -A to stage your changes before you commit them.
